Getting Started

DeathClient supports JavaScript plugins executed directly via the built-in browser engine. Plugins interact with the Java client using the global dc object.

Note: 3D rendering (onRender3D) is currently not officially supported for pure JS mods. You can hook into 2D rendering and tick updates.

// Example Plugin: AutoGG
var myMod = dc.registerMod({
    name: "AutoGG",
    description: "Says GG after a match.",
    category: "CHAT",
    isCheat: false,
    settings: []
});

myMod.onUpdate = function() {
    // Logic goes here
};

Mod Registration

Register a module using dc.registerMod(config). The config object accepts the following fields:

PropertyTypeDescription
nameStringRequired. The exact name of the module.
descriptionStringA short description shown in the click GUI.
categoryStringMOVEMENT, COMBAT, RENDER, WORLD, HUD, PLAYER, or MINIGAMES. Defaults to PLAYER.
isCheatBooleanMarks the module as a cheat (can be hidden on strict servers).
settingsArrayArray of setting objects (see below).

Settings API

Settings allow users to configure your script via the ClickGUI. They are passed in the settings array during registration.

Boolean Setting

{
    type: "boolean",
    name: "Spam",
    value: true
}

Number Setting

{
    type: "number",
    name: "Delay",
    value: 50,
    min: 0,
    max: 1000,
    step: 10
}

Mode Setting

{
    type: "mode",
    name: "Mode",
    value: "Normal",
    modes: ["Normal", "Aggressive", "Passive"]
}

To read a setting's value inside your mod callbacks, you can find it in the module's settings array:

var delaySetting = myMod.settings.find(s => s.name === "Delay");
var currentDelay = delaySetting.value;

Mod Callbacks

Once registered, you can assign functions to your module object to hook into game loops.

onEnable()Called when the module is toggled on.
onDisable()Called when the module is toggled off.
onUpdate()Called every client tick (usually 20 times per second).
onRender2D()Called every frame. Useful for drawing custom HUD elements.
onKey(keyCode)Called when a keyboard key is pressed while the mod is enabled.
onChat(message)Called when the player attempts to send a chat message.

Bridge API (dc.*)

The dc global object provides methods to interact with the game state.

dc.chat(msg)Prints a localized message to the user's chat box (not sent to server).
dc.sendChat(msg)Sends a message to the server, visible to other players.
dc.enableMod(name)Force enables a module.
dc.disableMod(name)Force disables a module.
dc.toggleMod(name)Toggles a module's state.
dc.setSetting(mod, set, val)Updates a setting programmatically.

Event System

Instead of tying logic to a specific mod, you can listen to global events using dc.on(eventName, callback).

tick

Fires every client tick. The payload includes basic player data.

dc.on("tick", function(payloadStr) {
    var player = JSON.parse(payloadStr);
    // player.x, player.y, player.z
    // player.health, player.maxHealth
    // player.onGround (boolean)
});

chat_receive

Fires when a chat message is received from the server.

dc.on("chat_receive", function(payloadStr) {
    var data = JSON.parse(payloadStr);
    if (data.message.includes("joined the game")) {
        dc.chat("Someone joined!");
    }
});

mod_enable & mod_disable

Fires when any module is toggled.

dc.on("mod_enable", function(payloadStr) {
    var data = JSON.parse(payloadStr);
    dc.chat("Mod enabled: " + data.name);
});

Data Persistence

Plugins can save and load data (like configs or stats) across client restarts using the built-in LocalStorage bridge.

dc.saveFile(key, val)Saves a string value under the given key.
dc.loadFile(key)Returns the saved string, or null if it doesn't exist.
dc.deleteFile(key)Deletes the stored key.
dc.listFiles(prefix)Returns an array of saved keys that start with the prefix.
 
// Example: Saving a kill count
var kills = parseInt(dc.loadFile("myplugin_kills")) || 0;
kills++;
dc.saveFile("myplugin_kills", kills.toString());

Color Codes

Use the section sign (§) to format text in your UI or chat messages.

Tip: In JavaScript string literals, write the section sign as '\u00a7' to avoid encoding issues. Example: '\u00a7aGreen Text'
